Проклятые поля

Неочевидная Простая Алгебра

July 8, 2019 Mike Permyakov

3 мин. чтения

На пути смелого искателя приключений возникли странные поля, излучавшие изумрудный свет.
Когда путник ступил на эти земли, он заметил странное жужжание исходившее от его часов. Стрелки вращаются с бешеной скоростью. Должно быть, это он замедлился.
Вдруг, он почувствовал голод и стал идти ещё медленнее.

Белое солнце палит. Кажется, что эта прогулка длится уже целую вечность.

Но… только взгляните… Это же граница странных полей. Зелёный свет начал растворяться, но голод всё ещё следовал за нашим путником…

На горизонте показалась огромная яблоня. Спасение.
Плотно перекусив, путешественник почувствовал прилив сил. Теперь он может продолжать свою дорогу.

Как мы можем воссоздать подобный опыт?

Давайте представим себе, что нам даны константа V, соответствующая нормальной скорости путника, и переменная Vc, соответствующая текущей скорости. Карта, по которой должен пройти главный герой будет иметь следующий вид:

field_starts   hunger_starts   field_ends    hunger_ends
(:             [               :)            ]

По преодолению каждой границы на карте, к нам будет поступать одно из следующих событий:

  • field_starts – вначале странных полей
  • field_ends – в конце странных полей
  • hunger_starts – когда путник почувствует голод
  • hunger_ends – когда путник наткнётся на яблоню

и, помимо того, мы знаем какого рода эффекты должны оказываться на текущую скорость путника:

  • когда он находится на странных полях, его скорость уменьшается в 5 раз
  • когда он начинает чувствовать голод, его скорость падает на 3 единицы

В таком случае, если мы должны построить систему, которая будет в каждый момент времени считать нам Vc, как должна выглядеть такая система?

Первая и самая простая идея, что приходит в голову – это использование прямых и обратных операций по требованию:

void start () {Vc = V;}
void update (Event event)
{
    if (event == field_starts) Vc /= 5;
    if (event == field_ends) Vc *= 5;
    if (event == hunger_starts) Vc -= 3;
    if (event == hunger_ends) Vc += 3;
}

Таким образом, работа завершена и выполнена она отвратительно.

Дело в том, что мы не можем комбинировать операции из разных классов, нарушая порядок применения прямых и обратных преобразований.

К примеру, пусть V равна 10. Значит, после путешествия, скорость путника будет составлять (((2 / 5) - 3) * 5) + 3 == -2.
Это плохой результат. Он не имеет никакого отношения к вышеизложенной истории.

АЧЁДЕЛАТЬТО??

Мы можем договориться о порядке, в котором могут применяться эффекты и собрать таблицу состояний, которая содержит информацию о активных на данный момент эффектах.

Рабочее решение может выглядеть примерно так:

void update(Event event)
{
    static map<event, pair<bool,function<(void)void> > > states {
        { field_effect,  {false, [&Vc](){Vc *= 5;} } },
        { hunger_effect, {false, [&Vc](){Vc += 3;} } },
    };

    static vector<event> effects {
        field_effect,
        hunger_effect,
    };

    Vc = V;
    for (auto& it : effects) {
        if (event == it) states[it].first = !states[it].first;
        if (states[it].first) states[it].second ();
    }
}

Таким образом, одинаковые наборы эффектов будут гарантировано приводить к одинаковому результату.

На этом всё. Спасибо за внимание.
